# 11.1 Projeto Garagem

## Objetivo da aula

Consolidar os conceitos do tutorial criando uma API paralela para uma garagem de veículos.

## Introdução

Este exercício transfere os padrões aplicados na livraria para outro domínio. A ideia é repetir o fluxo completo de modelagem, API e organização incremental por commits.

## Desenvolvimento

### 1. Criando o projeto

Crie um novo projeto a partir do template Django e nomeie o repositório como `garagem`.

Mensagem sugerida do primeiro commit:

```text
feat: criacao do projeto
```

### 2. Modelos básicos

Crie APIs para:

- `Acessorio`
- `Cor`
- `Modelo`

Faça um commit para cada etapa.

### 3. Modelo `Veiculo`

Crie o modelo `Veiculo` com:

- `ano`
- `preco`
- `modelo`
- `cor`
- `acessorios`

Depois, crie a API REST correspondente.

### 4. Diagrama obrigatório

Ao final, o diagrama `core.png` deve refletir a estrutura pedida no exercício.

## Hora do commit

```text
docs(secao-11): organiza exercicio do projeto garagem
```

## Prática

- Replique o fluxo completo da livraria em um novo domínio.
- Faça commits pequenos por etapa.
- Valide modelagem, serializers, views e rotas.

## Conclusão

Esse exercício fecha o ciclo do tutorial ao exigir autonomia para aplicar o conteúdo em outro contexto.
